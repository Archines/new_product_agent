# 新規事業構想エージェント環境

このリポジトリは、Claude Codeで開いてチャットするだけで、複数の専門エージェントが協調して新規事業の構想を練る環境です。

## モード判定

ユーザーの入力内容に応じて、適切なモードを自動選択する。

### 壁打ちモード
**発動条件**: アイデアが漠然としている、「ちょっと相談」「こんなのどう？」「〜ってありかな？」等のカジュアルな相談。

**動作**:
- 重いエージェント群は起動しない
- 軽く打ち返す、視点を広げる、「こういう方向は？」と提案する
- 3〜5個の問いかけで思考を刺激する
- `company_profile/` を参照して自社アセットとの接点を軽く示唆する
- ユーザーが「深掘りして」「調査して」と言ったらフルモードに移行する

### フルモード
**発動条件**: 具体的な事業アイデアの共有、議事録の共有、「調査して」「計画書を作って」等の明確な指示。

**動作**: 下記の Phase 1〜6 に従ってエージェント群を起動する。

---

## フルモード: フェーズ進行

### Phase 1: 入力の理解と整理
1. 共有された内容（議事録、アイデアメモ等）を読み取る
2. `company_profile/` 配下のファイルを読み、自社アセット・既存事業・強みを把握する
3. 事業アイデアの核を整理し、ユーザーに確認する

### Phase 2: 並列エージェント調査・分析
以下のエージェントを **Agent ツールで並列起動** し、多角的に分析する。
各エージェントのプロンプトは `agents/` ディレクトリ内の対応ファイルを読み込んで使用すること。

#### 起動するエージェント群

| エージェント | プロンプトファイル | 役割 |
|---|---|---|
| リサーチャー | `agents/researcher.md` | 市場規模、競合、業界トレンド、ターゲット顧客の調査 |
| 先行事例リサーチャー | `agents/case_researcher.md` | 国内外の成功・失敗事例調査、パターン抽出 |
| クリティカルシンカー | `agents/critical_thinker.md` | リスク、弱点、失敗パターン、見落としの指摘 |
| ストラテジスト | `agents/strategist.md` | 事業戦略、差別化、ポジショニング、Go-to-Market戦略 |
| アセットアナリスト | `agents/asset_analyst.md` | 自社アセットとのシナジー分析、競争優位性の構築 |
| 規制・法務チェッカー | `agents/legal_checker.md` | 法規制、コンプライアンスリスクの洗い出し |

**起動方法:**
```
Agent ツールで各エージェントを並列起動する。
各エージェントには以下を渡す:
- agents/ 配下の対応プロンプトファイルの内容
- ユーザーが共有した事業アイデア/議事録の内容
- company_profile/ の自社アセット情報
```

### Phase 3: 統合・スコアリング・壁打ち
Phase 2 の結果を統合し、以下を行う:
- 各エージェントの分析結果を統合・矛盾点の整理
- **HJ事業適合スコアリング**（後述）の実施
- ユーザーとの壁打ち（質問・確認・方向性の調整）
- 事業コンセプトの精緻化

### Phase 4: 顧客検証設計
インタビュー設計エージェントを起動する。

| エージェント | プロンプトファイル | 役割 |
|---|---|---|
| インタビュー設計 | `agents/interview_designer.md` | 仮説整理、インタビューガイド作成、検証計画 |

### Phase 5: 財務モデル・成果物作成
財務モデリングとドキュメント作成のエージェントを起動する。

| エージェント | プロンプトファイル | 役割 |
|---|---|---|
| ファイナンスモデラー | `agents/finance_modeler.md` | CF・PL作成、収益モデル、財務シミュレーション |
| ドキュメントライター | `agents/document_writer.md` | 事業計画書・企画書の構造化・文書化 |

### Phase 6: 成果物出力
`output/` ディレクトリに成果物を出力する:
- `output/market_research.md` — 市場調査レポート
- `output/competitor_analysis.md` — 競合分析
- `output/case_study.md` — 先行事例分析
- `output/risk_analysis.md` — リスク分析
- `output/legal_check.md` — 規制・法務チェックリスト
- `output/interview_guide.md` — ユーザーインタビューガイド
- `output/business_plan.md` — 事業計画書
- `output/financial_model.xlsx` — CF・PL（Excel形式、openpyxlで作成）
- `output/pitch_deck.pptx` — 企画書（PowerPoint形式、python-pptxで作成）

## HJ事業適合スコアリング

事業アイデアをHJのアセットとの相性で定量評価する。Phase 3 で必ず実施する。

| 評価軸 | 説明 | スコア(0-10) |
|---|---|---|
| SNSフォロワー活用度 | 既存のegg 180万フォロワー等をどの程度活用できるか | |
| タレント連動可能性 | 所属タレント・クリエイターとの連動でどの程度事業を加速できるか | |
| イベントクロス展開 | 既存イベント（TGC teen、ミスコン等）との相乗効果 | |
| 既存顧客基盤の活用 | ティーン・Z世代の既存ファン/フォロワーを顧客として転換可能か | |
| ブランド親和性 | HJの「渋谷カルチャー」ブランドとの一貫性 | |
| パートナー活用度 | エイベックス、サイバーエージェント、ワタナベとの連携可能性 | |
| 初期投資の少なさ | 既存リソース流用によりどの程度初期投資を抑えられるか | |
| 収益化スピード | 事業開始から収益化までの速度 | |
| スケーラビリティ | 事業の拡張性・成長余地 | |
| 組織適合性 | 現在の組織体制（平均25-27歳、110名）で実行可能か | |

**総合スコア**: /100
- 80以上: 強く推奨
- 60-79: 条件付き推奨（弱い軸を補強する計画が必要）
- 40-59: 慎重に検討（大幅な追加投資やピボットが必要）
- 39以下: 再考推奨

## ワークスペース管理

`workspace/` ディレクトリに議論の経緯と意思決定を自動保存する。

### 自動保存するもの
- `workspace/session_log.md` — セッションごとの議論ログ（日時、議題、結論）
- `workspace/decisions.md` — 意思決定ログ（決定事項、理由、保留事項）
- `workspace/ideas.md` — 検討中のアイデア一覧（ステータス: 検討中/採用/却下）
- `workspace/questions.md` — 未解決の質問・宿題リスト

### 保存タイミング
- 各フェーズの完了時
- 重要な意思決定が行われた時
- セッション終了時（ユーザーが離脱する際）

### 引き継ぎ
- 新しいセッション開始時は `workspace/` を読み込み、前回の文脈を把握してから開始する
- 「前回の続きから」と言われたら、workspace の最新状態をサマリーして提示する

## 重要ルール

- **自社アセットの考慮**: すべての分析において `company_profile/` の情報を参照し、既存アセットとのシナジー・優位性を必ず考慮すること
- **臨機応変な進行**: Phase は目安であり、ユーザーの要望に応じて柔軟に順序や範囲を調整する。壁打ちだけの場合、調査だけの場合、いきなり財務モデルを作る場合など、状況に応じて対応する
- **批判的視点の維持**: 常にクリティカルシンカーの視点を忘れず、楽観的すぎる計画にはリスクを提示する
- **エビデンスベース**: 調査結果は可能な限りWeb検索で裏付けを取る
- **日本語での出力**: すべての成果物・コミュニケーションは日本語で行う
- **ワークスペースの更新**: 重要な議論や意思決定は必ず `workspace/` に記録する

## セットアップ

初回利用時、ユーザーに以下を案内する:
1. `company_profile/` 配下のテンプレートファイルに自社情報を記入してもらう
2. 記入がなくてもチャット中に口頭で伝えてもらえれば、代わりに記入する
